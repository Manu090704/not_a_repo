¿Cuál es la diferencia entre Internet y la World Wide Web?
La www es una red de páginas en el internet y un consejo que pone las normas de la web. El Internet es una red de dispositivos conectados.
¿Cuáles son las partes de un URL?
Protocolo, dominio (ahora es una máquina virtual, antes era una computadora),ruta antes carpeta, antes página (recursos) y etiqueta hoy también son parámetros.
¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
  • GET: Se utiliza para consultar información de la base de datos.
  • HEAD: Se utiliza para consultar información de la base de datos solo el encabezado.
  • POST: Sube datos a la base de datos.
  • PUT: Modifica datos que ya se subieron a la base de datos. PATCH: Hace modificaciones especificas.
  • DELETE: Elimina datos de la base de datos.

POST
¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?
Se utiliza el método GET para desplegar la información sin hacer modificaciones.
servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?
Significa que la petición HTTP funciono correctamente, sin ningún problema ni error.
¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?
No, ya que no es un error tal cual del servidor, el error 404 significa que el recurso no fue encontrado, esto puede ser por ejemplo que el recurso a buscar ya fue eliminado del servidor lo cual no significa que este fallando si no que se hicieron modificaciones en los contenidos.
¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?
Sí, ya que los errores 500 significa que la solicitud fue válida pero que del lado del servidor no se logra procesar. Error en el servidor.

Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?
Que solo lo uses cuando sea urgente ya que no ayuda a tener un código limpio porque rompe con la jerarquía.
Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?
Porque puede afectar en la estética y experiencia del usuario ya que puede afectar al tiempo de espera para que cargue la página.
Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?
px nos puede ser muy útil con imágenes o bordes ya que es muy preciso. % este nos ayuda a ajustar con mayor flexibilidad. pt este si no es muy recomendable ya que en algunos navegadores ya no es compatible. Todos nos pueden ser útiles en ciertos eventos pero una buena práctica sería usar em ya que con esto el estilo se adapta al tamaño del display.
¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?
Porque el código es más pequeño y se puede cargar más rápido. Reduce el ancho de banda y hace que el tiempo sea menor.
